💡 全链交易流水字段与流转说明：

  • 处理网络：展示引擎捕获到该笔资金的具体区块链标识（如 TRON、BSC）。系统在底层已实现网络隔离，同哈希异链的数据不会发生串号或双花冲突。
  • 交易哈希：链上唯一转账凭证。文字为超链接状态，点击即可根据处理网络自动路由，跳转至对应的官方区块链浏览器进行区块高度查验。
  • 付款方 (From) 与 收款方 (To)：分别记录发币地址与您在系统配置的接单靓号。UI 层已根据出入金角色进行了蓝灰色块的视觉隔离。
  • 金额 (USDT)：此处显示的金额是扫块引擎通过读取您在【网络节点】中配置的“精度 (Decimals)”（如 6 或 18），自动执行换算除法后得出的真实法币数值。
  • 系统时间：该笔链上流水被 Workers 边缘计算节点拦截并写入底层 D1 数据库的绝对时间，前端已强制格式化为 Asia/Shanghai 北京时间。
  • 操作与清理：由于全网高频轮询会累积大量流水，系统提供了“批量删除”机制。为了避免 Cloudflare D1 存储容量溢出，建议定期对历史无争议的老旧账单进行批量勾选清理，但请务必保留近期的作为异步回调排查依据。

💡 回调路由参数说明：

  • 节点序数ID：系统 D1 数据库主键自动递增生成的唯一通道序列号，全局唯一。
  • 图标：节点的视觉徽标链接（如各发卡站的 Favicon），用于在多商户流转时直观区分不同的接货终端。
  • 名称：自定义的下游商户或系统代号。如“发卡A站”、“大客户B端”，方便多商户环境下的分类辨识。
  • CALLBACK_URL：外部接收充值入账通知的网关 API 接口。当您在【网络节点】中配置的任意一条激活区块链上扫出新订单时，本矩阵系统都会向此地址发起异步 POST 核心请求，数据包中会动态携带 network 字段。
  • APP_SECRET (核心变更)：通讯层安全验签密钥。**【极重要】**由于系统已升级为全动态矩阵架构，为了彻底阻断黑客利用“同一笔交易哈希在不同重放链”进行双花攻击，系统在哈希算法中引入了网络隔离。当前的签名拼接规则为：网络标识 + 交易哈希 + 到账金额 + APP_SECRET。下游接收端必须使用 SHA256 严格按此顺序组装验签！
  • 绑定靓号池：路由过滤器。可限定该回调通道仅接收指定收款地址的通知（支持英文逗号分隔多地址）；若填写 * 则默认不作过滤，放行全网所有收款地址的所有多链流水。
  • 添加日期：该通道规则进入系统的初始化时刻，系统已自动锁定并转换为 Asia/Shanghai 标准北京时间。
  • 备注：面向管理员的补充注释区，可备注对接负责人、结算账期或商户费率等附加说明。

    • 精准单链指定 (推荐)：如果在添加地址时指定了扫描网络（如选 BSC），系统将启动精准拦截，仅使用您在【网络节点】中配置的对应 RPC 巡检该地址，完美抗限流。
    • 智能自动识别 (全网并发)：若保持默认的自动识别，系统将读取您在【网络节点】中创建的所有激活网络，并根据地址格式进行动态派发：
      • 输入以 T 开头的地址：系统自动挂载所有自定义的 波场 (tron) 架构网络。
      • 输入以 0x 开头且长度为 42 的地址：系统自动并发挂载所有自定义的 以太坊兼容 (evm) 架构网络（如 ETH、BSC、Polygon、X Layer 等）。只要任意一条链有钱进入，均能秒级捕获。
      • 输入以 0x 开头且长度为 66 的地址：系统自动挂载 Aptos (aptos) 架构网络。
      • 输入以 UQ 或 EQ 开头的地址：系统自动挂载 TON (ton) 架构网络。
      • 输入常规 Base58 格式地址：系统自动挂载 Solana (solana) 架构网络。
  • 添加日期：录入系统的初始化时刻，自动转换为 Asia/Shanghai 标准时区。
  • 备注：面向管理员的附加文本区，可记录该地址的归集周期、所属业务线或负责人等。

安全鉴权设置

📖 系统部署与多链对接使用文档

架构特性： 本矩阵监控基于 Cloudflare Workers 边缘网络构建，网络节点、收款地址均已完成 KV 与驻留内存解耦。系统不仅免疫大规模 DDoS 攻击，且支持零代码动态扩容全新 EVM/Tron 兼容链。

1. 智能多模态靓号路由与单链过滤机制

系统在扫描时会实时加载【网络节点】中的全量配置，并根据【收款地址】中的指定策略进行双重路由判定：

- 精准单链绑定：若地址指定了扫描网络（如绑定了 BSC），扫块引擎执行时将启动精准拦截。在巡检其他链时该地址直接跳过，只调取 BSC 的 RPC 进行分析，这能节省 90% 的公共节点请求额度。
- 智能识别全网并发：若地址保持默认的 智能自动识别，引擎将根据地址特征将其并发挂载到所有相符架构的网络池中：
  • 粘贴以 T 开头的波场地址：自动并发出动全网所有内核类型标记为 tron 的网络。
  • 粘贴以 0x 开头且长度为 42 的多链地址：自动并发出动全网所有内核类型标记为 evm 的网络（如 ETH、BSC、Polygon、Base 等）。只要任意一条链有对应代币汇入，均能秒级触发入账。

2. 边缘高频轮询 (Cron Task) 配置

监控任务默认由全局 scheduled 定时器接管。系统在代码中配置了每分钟自动重置最新块高并向前追溯 50 个区块的防漏防错逻辑。若部署后发现无法自动巡检，请确保您在 Cloudflare 控制台的 Workers -> 设置 -> 触发器 -> 自动化 中正确挂载了 Cron 表达式：* * * * *。

3. 下游接货网关防双花安全验签 (PHP 示例)

由于系统支持多链混扫，**必须杜绝攻击者使用某一条链上的历史交易哈希去欺骗其他链网关的行为**。因此系统向您的 CALLBACK_URL 发送 POST 数据包时，加密签名的拼接顺序已升级为：网络标识 + 交易哈希 + 金额 + APP_SECRET。

<?php // 接收监控矩阵发来的原始 JSON 报文 $rawBody = file_get_contents('php://input'); $notifyData = json_decode($rawBody, true); if (!$notifyData) { die(json_encode(["code" => 400, "msg" => "无效请求"])); } // 1. 提取订单核心验签要素 $network = $notifyData['network']; // 动态拿到到账的网络标识，如 BSC / TRON $tx_hash = $notifyData['tx_hash']; // 区块链唯一交易哈希（请设为数据库唯一索引防止重放） $amount = $notifyData['amount']; // 过滤精度后的纯数字转账金额，如 100.5 $signReceived = $notifyData['sign']; // 矩阵网关生成的签名密文 $timestamp = $notifyData['timestamp']; // 矩阵秒级捕获时刻的时间戳 // 2. 严格按顺序构造本地签名 $app_secret = "您在回调路由面板中自定义设置的加密密钥"; $stringToSign = $network . $tx_hash . $amount . $app_secret; $calculatedSign = hash('sha256', $stringToSign); // 3. 校验对比 if ($calculatedSign !== $signReceived) { http_response_code(403); die(json_encode(["code" => 403, "msg" => "网关验证未通过：签名非法"])); } // 4. 验签通过，执行发货/充值业务逻辑 // TODO: 执行具体的卡密发放或商户余额入账 // ... echo json_encode(["code" => 200, "msg" => "SUCCESS"]); ?>
